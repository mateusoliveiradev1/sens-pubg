<task type="auto">
  <name>Task 3: Add explicit baseline update workflow</name>
  <files>scripts/update-benchmark-baseline.ts, package.json, src/core/benchmark-release-report.test.ts, docs/benchmark-runner.md</files>
  <read_first>scripts/run-benchmark.ts, tests/goldens/benchmark/synthetic-benchmark.baseline.json, tests/goldens/benchmark/captured-benchmark.baseline.json, docs/benchmark-runner.md</read_first>
  <action>
Add an explicit baseline update command, preferably exposed as `npm run benchmark:update-baseline -- --dataset synthetic|captured --reason "..." --affected-clips "..." --honesty-rationale "..."`.

Command rules:
- require dataset selector;
- require non-empty reason;
- require affected clips;
- require honesty rationale explaining why the new behavior is better or more honest;
- run the selected benchmark before writing the baseline;
- refuse to update if the benchmark itself structurally fails;
- write updated baseline JSON beside the dataset;
- append or emit a small markdown/update summary that can be committed with the baseline.

Do not let this command run implicitly from `benchmark:release`. The release gate should tell maintainers to run the update command when a regression is intentional.
  </action>
  <verify>npx vitest run src/core/benchmark-release-report.test.ts src/ci/benchmark-workflow.test.ts</verify>
  <acceptance_criteria>
    - `benchmark:update-baseline` exists in `package.json`.
    - The update script rejects missing reason, affected clips, or honesty rationale.
    - The update script reuses `createBenchmarkRegressionBaseline` or an equivalent stable baseline creator.
    - Docs explain when baseline updates are allowed.
  </acceptance_criteria>
  <done>Intentional regressions require an auditable baseline update rather than quiet expectation drift.</done>
