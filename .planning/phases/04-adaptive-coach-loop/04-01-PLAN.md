<threat_model>
Scope: authenticated outcome writes, revision audit rows, stored notes, and future memory signals.

Threats:
- HIGH: user records or revises outcomes for another user's analysis session. Mitigation: every read/write joins or filters `analysisSessions.id` with `analysisSessions.userId === auth().user.id`; tests cover cross-user denial.
- MEDIUM: outcome note becomes unsafe rendered text. Mitigation: store note as plain text, cap length, do not render HTML, rely on React escaping.
- MEDIUM: correction overwrites prior evidence and destroys auditability. Mitigation: correction creates a new row with `revisionOfId`/payload link and keeps previous row immutable.
- MEDIUM: malformed status/reason payload enters memory. Mitigation: pure validators reject unknown status, invalid reason combinations, overlong note, and `invalid_capture` without a reason.
</threat_model>

- `detectCoachOutcomePrecisionConflict`
