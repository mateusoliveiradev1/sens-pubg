<threat_model>
Scope: deterministic memory/tier decisions that will influence user training recommendations.

Threats:
- HIGH: global memory overrides strict compatible evidence and produces an unsafe strong recommendation. Mitigation: resolver uses strict compatible layer as primary; global layer only tie-breaks when strict memory is absent.
- HIGH: self-reported improvement immediately promotes apply_protocol. Mitigation: self-report is weak until compatible precision confirms it; tests prove `improved` alone cannot authorize apply.
- MEDIUM: stale compatible history dominates current weak evidence. Mitigation: hybrid recent-count/time-window filtering, visible stale/pending signals, and current clip usability gate.
- MEDIUM: repeated protocol failure loops the same advice. Mitigation: repeated failure adds blocker/new-hypothesis signal and lowers tier/aggressiveness.
</threat_model>

Encode adaptive coach decision rules.

Required behavior:
- `buildCoachPlan` always exposes one primary focus, up to two secondary focuses, and one next-block protocol for usable analyses.
- `apply_protocol` requires strong current evidence, no capture blocker, compatible memory without conflict, and recent validation/confirmed progress. One strong-looking current clip is not enough.
- Human outcome versus compatible-trend conflict blocks `apply_protocol` and returns `test_protocol` or `stabilize_block`.
- `stabilize_block` is selected when useful signal exists but variance, initial/oscillating trend, neutral outcomes, or repeated incomplete results make stronger action premature.
- Sensitivity loses to safer technique/validation when sensitivity and technique point in different directions unless strict compatible evidence supports sensitivity.
- A prior worsened protocol lowers aggressiveness and adds a new-hypothesis/short-validation reason.
- Repeating the same protocol requires a new hypothesis, new check, or new validation purpose in the protocol copy/id.
- Confirmed compatible progress asks to consolidate before changing variable.

Tests must cover strong current clip with no memory, confirmed progress, self-report conflict, repeated failure, invalid capture, sensitivity-vs-technique conflict, and max two secondary focuses.
