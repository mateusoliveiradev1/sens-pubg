Implement strict compatibility comparison.

Rules:
- Patch mismatch blocks.
- Weapon mismatch blocks.
- Scope mismatch blocks.
- Optic state mismatch blocks when both are known; missing dynamic optic state blocks precise trend.
- Stance, muzzle, grip, and stock mismatches block.
- Missing required metadata blocks.
- Distance missing, ambiguous, or outside a minimal tolerance blocks. Use a much stricter default than current history tolerance, such as 2m or a documented `STRICT_DISTANCE_TOLERANCE_METERS`.
- Weak or incomparable capture quality blocks precise trend. Compare `mastery.evidence.coverage`, `mastery.evidence.confidence`, `mastery.evidence.qualityScore`, `videoQualityReport.usableForAnalysis`, and sample size.
- Spray protocol mismatch blocks when window/duration/cadence evidence is available. Use conservative blockers for missing required spray window data when the current result has it.
- Strong sensitivity changes block or restart a line. Detect from `sensitivity.acceptanceFeedback`, current profile change, or stored precision metadata when available; keep the exact signal conservative and documented.

Tests must prove every required blocker emits a readable reason and that no trend math runs when `compatible=false`.

  <action>
Implement `resolvePrecisionTrend(input)` using current result plus prior sessions.

Trend behavior:
- zero compatible prior clips plus current usable clip creates `baseline`.
- one compatible prior clip plus current creates `initial_signal` unless blocked.
- two or more compatible prior clips plus current can reach `validated_progress`, `validated_regression`, or `oscillation`.
- use `mastery.actionableScore` as the primary score.
- keep `mastery.mechanicalScore` as secondary context.
- compute deltas from baseline to current and from recent compatible window to current.
- compute pillar deltas for control, consistency, confidence, and clip quality.
- use a conservative dead zone, for example 4 points, so small deltas become `oscillation`.
- require enough evidence for validated labels: compatible count >= 3 total including current, average coverage/confidence above strong thresholds, and no strong deterioration in confidence or clip quality.
- validated progress is blocked if any critical pillar deteriorates beyond its own tolerance.
- validated regression uses the same evidence bar as progress.
- recurring diagnoses explain trend only as explanatory context.
- `not_comparable` returns blockers and no false delta language.

Tests must cover baseline, initial signal, progress, regression, oscillation, weak evidence downgrade, critical pillar deterioration, and blocked trend.
  </action>

    - Two clips yield initial signal, not validated progress.
    - Small positive deltas stay oscillation.
    - Strong positive deltas with weak evidence do not become validated progress.
    - Regression requires sufficient evidence and a negative delta outside the dead zone.
    - Pillar deltas are always present when comparable.
