  <action>
Create `src/app/dashboard/dashboard-truth-view-model.ts` to convert `DashboardStats` into UI copy and state.

Export `buildDashboardTruthViewModel(stats)` returning:
- `nextActionTitle`
- `nextActionBody`
- `truthBadgeLabel`
- `scoreLabel`
- `trendTitle`
- `trendBody`
- `evidenceState` such as `strong`, `moderate`, `weak`, or `missing`
- optional `nextBlock` summary
- optional `arsenalFocus` summary

Rules:
- If latest mastery is `capture_again`, next action should be recapture/validate before training changes.
- If latest mastery is `inconclusive`, next action should be validation or a new comparable clip.
- If latest mastery is `testable`, next action should use coach next block when present.
- If latest mastery is `ready`, next action can say apply/validate protocol but must not say final/perfect/guaranteed.
- Positive raw trend deltas can be called progress only when recent trend evidence coverage/confidence meets the Phase 1 threshold. Otherwise call it "sinal para validar" or equivalent conservative PT-BR copy.
- Arsenal focus should remain contextual and should not dominate when evidence is weak.

Tests must cover capture-again, inconclusive, testable, ready, weak-trend-positive-delta, and strong-trend-positive-delta cases.
  </action>
  <verify>npx vitest run src/app/dashboard/dashboard-truth-view-model.test.ts</verify>
  <acceptance_criteria>
    - View model tests prove weak evidence does not produce improvement claims.
    - View model tests prove `ready` still uses validation language.
    - View model tests prove `testable` uses the next coach block when supplied.
    - View model output contains no `perfeito`, `garantido`, or equivalent final claims.
  </acceptance_criteria>
  <done>Dashboard copy decisions are deterministic and unit-tested outside the page component.</done>
