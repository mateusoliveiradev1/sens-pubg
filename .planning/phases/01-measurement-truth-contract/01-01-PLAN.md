  <action>
Add typed measurement truth contracts to `src/types/engine.ts` and a new resolver module `src/core/measurement-truth.ts`.

Required exported types:
- `SprayActionState = 'capture_again' | 'inconclusive' | 'testable' | 'ready'`
- `SprayActionLabel = 'Capturar de novo' | 'Incerto' | 'Testavel' | 'Pronto'`
- `SprayMechanicalLevel = 'initial' | 'intermediate' | 'advanced' | 'elite'`
- `SprayMechanicalLevelLabel = 'Inicial' | 'Intermediario' | 'Avancado' | 'Elite'`
- `SprayMasteryPillars` with numeric `control`, `consistency`, `confidence`, `clipQuality`
- `SprayMasteryEvidence` with `coverage`, `confidence`, `visibleFrames`, `lostFrames`, `framesProcessed`, `sampleSize`, `qualityScore`, `usableForAnalysis`
- `SprayMastery` with `actionState`, `actionLabel`, `mechanicalLevel`, `mechanicalLevelLabel`, `actionableScore`, `mechanicalScore`, `pillars`, `evidence`, `reasons`, `blockedRecommendations`

Add `readonly mastery?: SprayMastery` to `AnalysisResult`.

In `src/core/measurement-truth.ts`, export:
- `resolveMeasurementTruth(input)` where input includes at least `metrics`, `trajectory`, `sensitivity`, optional `videoQualityReport`, optional `diagnoses`, optional `coachPlan`, optional `subSessions`.
- `formatDiagnosisTruthLabel(type)` that maps `underpull` to `spray subindo`, `overpull` to `spray descendo demais`, `excessive_jitter` to `tremida lateral`, `horizontal_drift` to `desvio lateral`, `inconsistency` to `spray irregular`, `late_compensation` to `reacao atrasada ao recuo`, and `inconclusive` to `leitura inconclusiva`.

Scoring rules:
- Keep existing `metrics.sprayScore` as the base `mechanicalScore`; do not change `calculateSprayMetrics`.
- Pillar `control` should use current mechanical control signals, at minimum normalized `verticalControlIndex` and `horizontalNoiseIndex`/`linearErrorSeverity`.
- Pillar `consistency` should use `metrics.consistencyScore`.
- Pillar `confidence` should use metric quality/tracking confidence and coverage; prefer `metrics.metricQuality.sprayScore` when present, otherwise derive from `trajectory`.
- Pillar `clipQuality` should use `videoQualityReport.overallScore` when present, otherwise a neutral usable default that does not manufacture strong evidence.
- If `usableForAnalysis === false`, `sensitivity.tier === 'capture_again'`, coverage < 0.6, or confidence < 0.6, set action state to `capture_again` or `inconclusive`, cap `actionableScore` below the testable range, and add blocked recommendation reasons.
- If evidence is moderate, map to `testable`.
- Map to `ready` only when existing strong signals support it: `sensitivity.tier === 'apply_ready'` or `coachPlan.tier === 'apply_protocol'`, strong evidence, and repeated compatible samples (`subSessions.length >= 3` or history-enriched sensitivity already says apply-ready).
- Do not expose raw formula weights in any `reasons` string.

Export the resolver from `src/core/index.ts`.
