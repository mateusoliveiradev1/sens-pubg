<threat_model>
Scope: quota ledger, save action, and analysis result entitlement/quota return shape.

Threats:
- HIGH: client bypasses quota by calling `saveAnalysisResult` directly. Mitigation: save action resolves access/quota server-side every time.
- HIGH: concurrent tabs exceed free/pro quota. Mitigation: reservation/finalization uses transaction-safe unique idempotency keys and period counters.
- HIGH: client supplies period/remaining quota. Mitigation: all period and quota values come from resolver/Stripe/server UTC only.
- MEDIUM: technical failure or weak-capture inconclusive consumes quota unfairly. Mitigation: finalization classifies billable/non-billable with reason codes and voids reservations.
- MEDIUM: deleting history refunds quota silently. Mitigation: deletion does not refund; support adjustment is a distinct audited ledger entry.
</threat_model>
