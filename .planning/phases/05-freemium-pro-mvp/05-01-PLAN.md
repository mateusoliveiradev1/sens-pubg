`resolveProductAccess` should accept normalized facts:
- subscription state and periods;
- checkout pending state;
- manual grants;
- suspension/fraud/chargeback;
- quota summary;
- feature catalog;
- server-side flags;
- now.

Return:
- `effectiveTier`;
- `accessState`;
- `source`;
- `billingStatus`;
- `quota`;
- `features`;
- `blockers`;
- period/expiry fields;
- safe audit references.

Resolver precedence:
1. suspension/fraud/chargeback wins;
2. Stripe active/founder/canceling grants Pro until paid period end;
3. past_due grants during the 3-day grace period only;
4. active manual grants apply when Stripe does not;
5. canceled/expired/free do not grant Pro;
6. quota can block saves without deleting history/billing access.

Add `resolveMonetizationFlags` so server flags fail closed by default and safe mode has explicit behavior: preserve confirmed paid access, block new checkout if configured, pause quota consumption only when explicitly enabled, and show a degraded-state blocker.
