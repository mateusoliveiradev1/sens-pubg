<threat_model>
Scope: monetization/product analytics event recorder and instrumentation points.

Threats:
- HIGH: analytics stores raw clip, filename, trajectory, full analysis payload, private notes, or payment/identity documents. Mitigation: typed event allowlist plus metadata sanitizer that drops unknown keys and rejects prohibited values in tests.
- HIGH: client emits trusted activation/upgrade/billing events directly. Mitigation: critical events are emitted from server actions, quota ledger, projection policy, and webhook fulfillment only.
- MEDIUM: legacy analyses are counted as Phase 5 activation. Mitigation: first activation requires first useful saved analysis after instrumentation timestamp.
- MEDIUM: analytics failure blocks save or checkout. Mitigation: event writes are best effort with explicit error audit path where appropriate, but never mutate entitlement truth.
- MEDIUM: event names drift across UI and server code. Mitigation: central event union and helper functions.
</threat_model>
