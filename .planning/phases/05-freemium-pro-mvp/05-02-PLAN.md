<threat_model>
Scope: checkout creation, Stripe webhook ingestion, subscription state mutation, and portal sessions.

Threats:
- HIGH: user tampers with price/currency/plan/client entitlement. Mitigation: action accepts only a narrow intent, maps to server-owned catalog, and tests reject arbitrary payloads.
- HIGH: success URL grants Pro before trusted webhook. Mitigation: success page/action only reads resolver state; Pro state changes happen in webhook fulfillment.
- HIGH: fake or replayed webhook grants Pro. Mitigation: raw-body signature verification, unique event table, idempotent processing, and metadata/customer/subscription validation.
- HIGH: out-of-order subscription events reopen Pro after cancellation/suspension. Mitigation: lifecycle resolver uses Stripe period timestamps, monotonic processing rules, and suspension precedence.
- MEDIUM: checkout spam or duplicate sessions. Mitigation: per-user/IP rate limit plus internal checkout attempts and Stripe idempotency keys.
</threat_model>
