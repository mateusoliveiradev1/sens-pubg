<threat_model>
Scope: admin/support billing surface, manual grants, suspensions, reconciliation, flags, beta cohort operations, and runbooks.

Threats:
- HIGH: support can grant/revoke/suspend Pro. Mitigation: admin-only mutations; support read/add-note only.
- HIGH: manual grant abuse or invisible flag changes. Mitigation: required reason/category/dates/actor/target, audit logs, analytics event, and runbook.
- HIGH: safe mode silently grants Pro to everyone. Mitigation: safe mode behavior is explicit and covered in resolver/flag tests.
- HIGH: webhook signature flag disabled in production. Mitigation: production flag validation rejects disabling signature enforcement.
- MEDIUM: creator beta creates undisclosed material-benefit promotion. Mitigation: cohort/creator helper requires disclosure state before paid invite code.
- MEDIUM: refund/dispute handling deletes or falsifies history. Mitigation: suspension/revocation only affects access; history stays intact.
</threat_model>
