<threat_model>
Scope: analysis contract versioning, spray validity, blocker reasons, and pre-tracking analysis gating.

Threats:
- HIGH: invalid clips are analyzed as if they were valid sprays. Mitigation: `detectSprayValidity` returns explicit invalid/blocker reasons and the decision ladder blocks downstream strong decisions.
- HIGH: new ladder breaks old saved history. Mitigation: add optional versioned fields and keep old `SprayActionState` readable through a compatibility mapping.
- HIGH: invalid/inconclusive clips count as paid useful quota. Mitigation: permission matrix sets `countsAsUsefulAnalysis=false` below `usable_analysis`.
- MEDIUM: recapture states feel like app failure. Mitigation: blocked results carry reason codes and recapture guidance.
- MEDIUM: public or UI copy implies perfect sensitivity or guaranteed improvement. Mitigation: tests must reject perfect/guaranteed/final claim wording.
</threat_model>

  <action>
Extend spray window detection with `SprayValidityReport`.

Keep `detectSprayWindow(frames, options)` for existing callers, but implement it through a new `detectSprayValidity(frames, options)` helper.

`detectSprayValidity` must return:
- `valid: boolean`
- `decisionLevel: AnalysisDecisionLevel`
- `window: SprayWindowDetection | null`
- `blockerReasons: AnalysisBlockerReasonCode[]`
- `trimmedReasons: AnalysisBlockerReasonCode[]`
- `recaptureGuidance: string[]`
- `frameCount`
- `shotLikeEvents`
- `confidence`

Add concrete heuristics:
- fewer than 6 frames -> `too_short`
- fewer than 2 shot-like events -> `not_spray_protocol`
- no visible reticle in every sampled frame -> `crosshair_not_visible`
- a frame-to-frame displacement above 28 px inside a short sequence -> `flick`
- two or more high displacement jumps above 40 px -> `target_swap`
- a leading or trailing discontinuity above 55 px -> `hard_cut`

Use the new reason codes without deleting the existing `SprayWindowDetection` fields.
  </action>
