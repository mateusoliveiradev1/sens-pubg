## Hardening update

Validated later on: 2026-04-13

### Current verified status

Commands re-run after W00 hardening:

```bash
npx vitest run
if (Test-Path '.next') { Remove-Item '.next' -Recurse -Force }
npm run build
npx playwright test
```

Observed result:

- `npx vitest run`: 5 files passed, 33 tests passed
- `npm run build`: passed when run in isolation after cleaning `.next`
- `npx playwright test`: 21 tests passed

### Root causes confirmed

1. Unit tests

- `coach-engine.test.ts` was brittle and expected exact string equality where the engine now returns richer guidance
- `sensitivity-engine.ts` omitted DPI from the recommendation reasoning payload

2. E2E failures

- header navigation used `menubar/menuitem` roles on normal links, which broke role-based selectors and harmed semantics
- `/analyze` test assumed the dropzone was always present, but the real flow correctly gates analysis behind setup completion
- header/mobile navigation was missing `Perfil`
- login accessibility behavior was unstable because dev CSP blocked Next fallback scripts in `npm run dev`, preventing reliable client hydration in the login form

3. Build instability

- previous build failures were reproducible against stale runtime artifacts and concurrent execution
- isolated build with clean `.next` completed successfully

### Fixes applied

- relaxed dev-only CSP in `src/middleware.ts` so Next development fallback scripts can load
- improved header and mobile-nav semantics and added `Perfil`
- aligned E2E expectations with actual setup-gated analyze flow
- made login provider buttons keyboard-focusable even before terms acceptance, while still blocking login action in code until terms are accepted
- fixed brittle unit expectations and completed missing reasoning context

### Remaining note

- build is green in isolated execution; if future automation runs build concurrently with active dev servers or stale artifacts, the first step should still be to kill the orphan server and clean `.next`
